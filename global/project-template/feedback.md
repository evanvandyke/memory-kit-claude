# [Project] — Feedback

## How to write & repair this file
- **No `Why:` narratives.** State the rule; fold the reasoning in only if needed. No incident replay.
- **≤150 characters per entry** (content only, excluding the number prefix).
- **Forward postures only** — what TO do, not what went wrong. A rule framed as avoidance plants the avoided behavior.
- **Numbered sequentially** (1, 2, 3…); renumber to close gaps after a purge.
- **Max 15 entries** — quality over accumulation.
- **Purge on overflow:** a 16th forces dropping one to stay ≤15, surfaced never silent (say which and why). Purge priority: graduated-to-global → superseded → least-applicable.

One file, a flat numbered list (no category headers). Do not create `feedback_*.md` siblings.
